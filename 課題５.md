## １．PHPのインストール

要求仕様の中に「７系をインストール」と記載があるため  
今自分の使用しているディストリビューションに何系のPHPが入っているのか調べる  

使用したコマンドと出力結果

標準搭載のバージョンが8.1なことが分かった。  

標準のリポジトリから取得することが出来ない為、サードパーティのリポジトリを入れ込む  
後述する参考webサイトに掲載されていたリポジトリが、信頼性が高いとされているPPAだった為  
記事に書かれているリポジトリをそのまま入れ込むことにする  

参考にしたwebサイト  
(https://blog.apar.jp/linux/14381/)  
(https://hitori-josys.me/entry/ubuntu-install-php74)  

使用したコマンド  
sudo apt update ←パッケージリストを最新化し依存関係の不整合を防ぐ  

sudo apt install software-properties-common　←外部リポジトリを管理するコマンド（add-apt-repository）を使用可能にする  

sudo add-apt-repository ppa:ondrej/php ←リポジトリをインストールする  


・リポジトリのインストールが無事に終わった為、色々コマンドを打って試行錯誤する  

apt show php ←このコマンドだと接続可能なリポジトリにある最新のバージョンが表示される  

apt list php7.4 ←バージョン指定をしてリスト表示を行えばリポジトリに7系があることが分かる  

インストールコマンドが　apt install php だとバージョンの指定が出来ない為、バージョン名を明示してインストールを実施  

使用したコマンド  
sudo apt install php7.4  

バージョン確認のコマンドを入力し、無事に7系がインストール出来たことを確認

## ２．PHPinfoの情報表示（webブラウザにて）  
今までnanoエディターでhtmlの中身の入力をしていたが、  
webサイトの内容を色々と精査していると、viエディターの方が汎用性が高い  
という情報が多く見受けられた為、今回はviエディターを使用  

使用したコマンド  
sudo vi /var/www/kawaguchi-hbtask.local/index.html/phpinfo.php  

上記のコマンドは間違っていて、画像のようなエラーが生じた  
<img width="504" height="678" alt="image" src="https://github.com/user-attachments/assets/2d7a73d2-358e-4de6-93e3-08cdb505bb28" />  

Readonly ＝　書き込み不可
その為、一度ターミナル画面に戻る為に「：wq」を入力したがそもそもこのエディターに入る際に使用した  
index.html/phpinfo.pho　というファイルが存在しない為、保存して戻ることは出来なかった  

wq ＝　write 書き込み  quit 終了  

wqで戻れなかった為、「：q」で戻ろうとした際には  
<img width="501" height="680" alt="image" src="https://github.com/user-attachments/assets/efc5cd77-c3c2-43ee-ae5b-0936cbdf2ca5" />  

このようなエラー表示が出た  
これは、ひらたく言えば保存せずに閉じることはできないよ　という表示らしい  

その為、強制実行のコマンドを入力しターミナル画面に戻ることに  
「：q!」  

今度こそ、正しいパスを入力しエディター画面に入る  
sudo vi /var/www/kawaguchi-hbtask.local/phpinfo.php  

エディター内の入力内容  
<?php  
phpinfo();  
?>  

今度はきちんと「：ｗｑ」でエディターを閉じることが出来た  

その後ファイアフォックスで自分の名字-hbtask.local/phpinfo/phpにアクセスし、無事に画面に表示することが出来た
