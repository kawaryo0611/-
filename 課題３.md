-timedatectl  
-timedatectl set-timezone Asia/Tokyo  
-timedatectl status

- NTPサーバとの時刻同期  
-ntp1.jst.mfeed.ad.jp  
-ntp2.jst.mfeed.ad.jp  
-ntp3.jst.mfeed.ad.jp

それっぽいマニュアルがないか試しに入力したが見つからなかった為ドキュメント探し。

chronyをインストールするコマンドを実行  
sudo apt install chrony  

NTPサーバとの時刻合わせのコマンドを実行  
ntpdate -q ntp1.jst.mfeed.ad.jp  
ntpdate -q ntp2.jst.mfeed.ad.jp  
ntpdate ntp3.jst.mfeed.ad.jp  

ntp1,ntp2は確認だけ行う為にオプションのqを付けたがntp3は除いている

## ファイアウォールの有効化

まずは別サイトや公式ドキュメントは見ず、課題文に記載されている「firewalld」をそのまま入力し実行  
ターミナルからの応答でインストールコマンドが判明、そしてそれを実行  
  sudo apt install firewalld  
  
ダウンロードはしたが、実装方法が分からない為ひとまずman firewalldと入力  
全文目を通したがどの部分を参考にすべきか不明な為、より簡易的な表示を出せるコマンドに変更  
  firewalld --help  
  
コマンドをルート権限にする必要があったので打ち直す  
  sudo firewalld --help

  firewall-cmd --state

## SELinuxの無効化
未経験IT転職者の私にはSEという文字を見るとシステムやエンジニアという文字が連想される。  
その発想でいくと今回のこの項目はシステムエンジニアリナックス、かと思いきや全然違った（余談）  
SELinux  =  Security-Enhanced Linux 
この「SELinux」自体が先述のファイアウォール等の代わりになるわけではなく  
セキュリティ制御の追加機能、のようなもの  

状態確認の為のコマンド、getenforceを入力しましたが  
私が使っているバージョンのUbuntuには標準搭載されていないようだったのでダウンロードから行う  
インストールする為に実施したコマンド  
sudo apt update  
sudo apt install policycoreutils selinux-utils selinux-basics  

ここで確認の為のコマンドを入力  
sestatus  

disableの表示が出たが、そもそもインストールしてから有効化にしていなかったので  
sudo selinux-activate  

すると画面にrebootを要求する表示が出た為再起動  
もう一度、確認コマンドを入力したが  
selinux　は disableのままだった  

そのため次は強制モードというのを取り入れてもう一度再起動  
sudo selinux-config-enforcing  

すると次のような画面になり起動が中断される  
<img width="728" height="459" alt="image" src="https://github.com/user-attachments/assets/e4cc5d8b-437d-4ae4-8529-028a0efafdbd" />  

元々参考にしていたWebサイト(https://ja.unixlinux.online/ou/1007002300.html)  の中に  
UbuntuはSELinuxの代わりにAppArmorが入っている。この機能はSELinuxと相性が悪い  
という記載があったので恐らくこれだと思い対策をする  
↓以下、対策内容  

選択肢として、SELinuxを  
１　無効化  
２　記録のみ  
にする必要があったが、そもそもこの画面では入力することができなかったので  
仮想マシンを再起動し、OSを起動する前のメニューから編集を行うことにする  
ブート画面でF12を押し、カーネルパラメータを変更するためにEを押す  
E　= Edit（編集）  
関連単語、ブートローダー、GRUB  

<img width="627" height="466" alt="image" src="https://github.com/user-attachments/assets/414902cf-c2d1-4361-b84e-0baf1dbabc65" />  

GRUBの記述の中のLinuxの行を編集  
quiet splash　のすぐ後に「selinux=0」のコマンドを入力し、ブート選択（f10）  
無事に起動することができた  

接続後、設定を変更しにいく（SELinuxの設定を変える）  
使ったコマンド  

　nano  
どのような時に使用するか  
-設定ファイルを書き換える  
-サーバーの設定を編集する  
-ファイルの中身をちょっと直す  

今回は上記の nano + etc/selinux/config を使用した  
設定完了後rebootを行い設定の再確認、disabledに変わっているのを確認し今回の課題は終了
